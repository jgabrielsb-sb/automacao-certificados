Hexagonal Architecture Guide
============================

This guide explains how to properly implement use cases following hexagonal architecture
(ports and adapters) principles in this project.

Core Principles
---------------

1. **Use Cases depend on Ports (Interfaces), not Adapters**
   - Use cases should only know about abstractions
   - Concrete implementations are injected at runtime

2. **Ports define contracts in the core layer**
   - Ports are interfaces that define what operations are needed
   - They live in ``core/interfaces/``

3. **Adapters implement ports**
   - Adapters provide concrete implementations
   - They live in ``adapters/`` or ``infra/``

4. **Use Cases orchestrate business logic**
   - They coordinate between ports to achieve business goals
   - They don't contain infrastructure concerns

Structure Overview
------------------

::

   core/
   ├── interfaces/          # Ports (what we need)
   ├── models/              # Domain models and DTOs
   └── exceptions/          # Domain exceptions
   
   application/
   └── use_cases/           # Business logic orchestration
   
   adapters/                # Implementations of ports
   infra/                   # Infrastructure adapters

Implementing a Use Case
-----------------------

Step 1: Define the Port (Interface)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

First, create a port in ``core/interfaces/`` that defines what operations
the use case needs:

.. code-block:: python

   from abc import ABC, abstractmethod
   from automacao_certificados.selenium_automations.core.models import CertificateToDownload
   
   class CertificateRepositoryPort(ABC):
       """
       Port for retrieving certificates that need to be downloaded.
       
       This port abstracts the source of certificates, allowing different
       implementations (PPE API, database, file, etc.)
       """
       
       @abstractmethod
       def get_certificates_to_download(self) -> list[CertificateToDownload]:
           """
           Retrieve all certificates that need to be downloaded.
           
           Returns:
               List of certificates to download.
           
           Raises:
               CertificateRepositoryException: If retrieval fails.
           """
           pass

Step 2: Define Input/Output Models
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Create DTOs for the use case in ``core/models/application/use_cases/``:

.. code-block:: python

   from pydantic import BaseModel
   from automacao_certificados.selenium_automations.core.models import (
       CertificateToDownload,
       WorkflowOutput,
   )
   
   class DownloadCertificatesUseCaseInput(BaseModel):
       """Input for the download certificates use case."""
       pass  # Add parameters if needed
   
   class DownloadCertificatesUseCaseOutput(BaseModel):
       """Output for a single certificate download attempt."""
       certificate: CertificateToDownload
       error_selection: str | None = None
       workflow_output: WorkflowOutput

Step 3: Implement the Use Case
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

The use case depends only on ports:

.. code-block:: python

   from automacao_certificados.selenium_automations.core.interfaces import (
       CertificateRepositoryPort,
       WorkflowPort,  # Assuming this exists
   )
   from automacao_certificados.selenium_automations.core.models import (
       DownloadCertificatesUseCaseInput,
       DownloadCertificatesUseCaseOutput,
   )
   from automacao_certificados.selenium_automations.core.exceptions import (
       DownloadCertificatesUseCaseException,
   )
   
   class DownloadCertificatesUseCase:
       """
       Use case for downloading certificates from multiple sources.
       
       This use case orchestrates:
       1. Fetching certificates that need to be downloaded
       2. Selecting the appropriate workflow for each certificate
       3. Executing the workflow to download and process certificates
       4. Returning results with success/error information
       
       Attributes:
           certificate_repository: Port for retrieving certificates.
           workflow_selector: Service for selecting appropriate workflows.
       """
       
       def __init__(
           self,
           certificate_repository: CertificateRepositoryPort,
           workflow_selector: WorkflowSelector,
       ):
           if not isinstance(certificate_repository, CertificateRepositoryPort):
               raise ValueError(
                   'certificate_repository must be a CertificateRepositoryPort'
               )
           
           if not isinstance(workflow_selector, WorkflowSelector):
               raise ValueError('workflow_selector must be a WorkflowSelector')
           
           self.certificate_repository = certificate_repository
           self.workflow_selector = workflow_selector
       
       def execute(
           self,
           input: DownloadCertificatesUseCaseInput
       ) -> list[DownloadCertificatesUseCaseOutput]:
           """
           Execute the certificate download process.
           
           Args:
               input: Use case input (currently empty, but extensible).
           
           Returns:
               List of download results, one for each certificate processed.
           
           Raises:
               DownloadCertificatesUseCaseException: If the process fails.
           """
           try:
               certificates = self._fetch_certificates()
               return self._process_certificates(certificates)
           except Exception as e:
               raise DownloadCertificatesUseCaseException(
                   f"Failed to download certificates: {e}"
               ) from e
       
       def _fetch_certificates(self) -> list[CertificateToDownload]:
           """Fetch certificates that need to be downloaded."""
           return self.certificate_repository.get_certificates_to_download()
       
       def _process_certificates(
           self,
           certificates: list[CertificateToDownload]
       ) -> list[DownloadCertificatesUseCaseOutput]:
           """Process each certificate through its workflow."""
           results = []
           
           for certificate in certificates:
               result = self._process_certificate(certificate)
               results.append(result)
           
           return results
       
       def _process_certificate(
           self,
           certificate: CertificateToDownload
       ) -> DownloadCertificatesUseCaseOutput:
           """Process a single certificate."""
           try:
               workflow = self.workflow_selector.get_workflow(
                   certificate.cnpj,
                   certificate.document_type
               )
               workflow_output = workflow.run(certificate.cnpj)
               
               return DownloadCertificatesUseCaseOutput(
                   certificate=certificate,
                   error_selection=None,
                   workflow_output=workflow_output,
               )
           except WorkflowSelectorException as e:
               return DownloadCertificatesUseCaseOutput(
                   certificate=certificate,
                   error_selection=str(e),
                   workflow_output=WorkflowOutput(),
               )

Step 4: Implement the Adapter
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Create an adapter that implements the port:

.. code-block:: python

   from automacao_certificados.selenium_automations.core.interfaces import (
       CertificateRepositoryPort,
   )
   from automacao_certificados.selenium_automations.core.models import (
       CertificateToDownload,
   )
   from automacao_certificados.selenium_automations.core.exceptions import (
       CertificateRepositoryException,
   )
   
   class PPECertificateRepository(CertificateRepositoryPort):
       """
       Adapter that retrieves certificates from PPE API.
       
       This is a concrete implementation of CertificateRepositoryPort
       that uses the PPE API as the data source.
       """
       
       def __init__(self, ppe_api_requester: PPEAPIRequester):
           self.ppe_api_requester = ppe_api_requester
       
       def get_certificates_to_download(self) -> list[CertificateToDownload]:
           """Retrieve certificates from PPE API."""
           try:
               return self.ppe_api_requester.get_certificates_to_download()
           except Exception as e:
               raise CertificateRepositoryException(
                   f"Failed to retrieve certificates from PPE API: {e}"
               ) from e

Step 5: Create Dependency Injection Container
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Instead of wiring dependencies directly in `main.py`, create a centralized
dependency injection container:

.. code-block:: python

   # infra/dependency_injection.py
   from automacao_certificados.config import settings
   
   def create_ppe_api_requester() -> PPEAPIRequester:
       """Create PPE API requester with default configuration."""
       return PPEAPIRequester(
           http=HttpxClient(),
           api_key=settings.ppe_api_key
       )
   
   def create_certificate_repository() -> CertificateRepositoryPort:
       """Create certificate repository adapter."""
       ppe_api = create_ppe_api_requester()
       return PPECertificateRepository(ppe_api_requester=ppe_api)
   
   def create_workflow_selector() -> WorkflowSelector:
       """Create workflow selector with default configuration."""
       municipio_getter = ReceitaAPIMunicipioGetter(...)
       return WorkflowSelector(municipio_api_requester=municipio_getter)
   
   def create_download_certificates_use_case() -> DownloadCertificatesUseCase:
       """Create use case with all default dependencies."""
       return DownloadCertificatesUseCase(
           certificate_repository=create_certificate_repository(),
           workflow_selector=create_workflow_selector(),
       )

Step 6: Use in Main Application
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Now your `main.py` is clean and simple:

.. code-block:: python

   # main.py
   from automacao_certificados.selenium_automations.infra.dependency_injection import (
       create_download_certificates_use_case,
   )
   
   # All dependencies created automatically with defaults
   use_case = create_download_certificates_use_case()
   
   # Execute
   results = use_case.execute(DownloadCertificatesUseCaseInput())

Key Benefits
-----------

1. **Testability**: Easy to mock ports for unit testing
2. **Flexibility**: Can swap implementations without changing use case
3. **Separation of Concerns**: Business logic separate from infrastructure
4. **Maintainability**: Clear boundaries and responsibilities

Common Patterns
---------------

Use Case Naming
~~~~~~~~~~~~~~~

- Use cases are named with verbs: ``DownloadCertificatesUseCase``
- Methods are named with verbs: ``execute()``, ``process()``, ``fetch()``

Port Naming
~~~~~~~~~~~

- Ports end with ``Port``: ``CertificateRepositoryPort``
- They describe what they provide, not how

Adapter Naming
~~~~~~~~~~~~~~

- Adapters are named by their implementation: ``PPECertificateRepository``
- They indicate the concrete technology/source

Error Handling
~~~~~~~~~~~~~~

- Use case catches domain exceptions and wraps them
- Adapters catch infrastructure exceptions and convert to domain exceptions
- Each layer has its own exception types

Testing
-------

With this structure, testing becomes straightforward:

.. code-block:: python

   def test_download_certificates_use_case():
       # Mock the port
       mock_repository = Mock(spec=CertificateRepositoryPort)
       mock_repository.get_certificates_to_download.return_value = [
           CertificateToDownload(cnpj="123", document_type=...)
       ]
       
       mock_selector = Mock(spec=WorkflowSelector)
       mock_selector.get_workflow.return_value = MockWorkflow()
       
       # Create use case with mocks
       use_case = DownloadCertificatesUseCase(
           certificate_repository=mock_repository,
           workflow_selector=mock_selector,
       )
       
       # Execute and verify
       results = use_case.execute(DownloadCertificatesUseCaseInput())
       assert len(results) == 1
